# Bé Tiền Admin Console — Production Deploy Runbook

## Scope

Deploy the Phase 4.2.5 Admin Observability Console to `https://admin.betien.vn` with HTTPS, restricted CORS, API rate limits, and rollback instructions.

## Preconditions

- DNS `A` record for `admin.betien.vn` points to the production VPS.
- Caddy is installed and built with a `rate_limit` module.
- `infra/caddy/Caddyfile.admin` has been merged into `/etc/caddy/Caddyfile`.
- Backend environment contains:
  - `ADMIN_JWT_SECRET` generated by `openssl rand -hex 32`.
  - `ADMIN_JWT_EXPIRY_MINUTES=60`.
  - `ADMIN_ALLOWED_ORIGIN=https://admin.betien.vn`.
  - `ADMIN_API_RATE_LIMIT_PER_MINUTE=100`.
  - `ADMIN_REDIS_URL=redis://localhost:6379/1`.
  - `DATABASE_URL` for production Postgres.
- One-time seed variables are available only during first deploy: `INITIAL_ADMIN_EMAIL` and `INITIAL_ADMIN_PASSWORD`.

## Step-by-step deploy

1. SSH to the production host and enter the repo directory.
2. Pull the target release branch and verify the commit SHA.
3. Validate Caddy before touching the running process:
   ```bash
   caddy validate --config /etc/caddy/Caddyfile
   ```
4. Run the admin deploy script:
   ```bash
   ./scripts/deploy_admin.sh
   ```
5. Confirm health:
   ```bash
   curl -fsS https://admin.betien.vn/health
   ```
6. Run the smoke test checklist below.
7. Remove one-time seed password variables from the host environment after first login succeeds.

## Smoke test checklist

1. Open `https://admin.betien.vn` and confirm the SPA loads.
2. Directly open `https://admin.betien.vn/login` and confirm the login page renders.
3. Login with the seeded admin and confirm first login redirects to change password.
4. Change password and confirm dashboard access.
5. KPI cards show production data, not mock placeholders.
6. All dashboard charts render without console errors.
7. User table search, tier filter, status filter, sorting, and pagination work.
8. Open a user detail modal and confirm the default view masks PII.
9. Use the reveal action only when needed and verify an audit log entry records `pii_revealed`.
10. Logout, then open a protected URL directly and confirm it redirects back to login.

## Rollback procedure

1. Capture current logs for diagnosis:
   ```bash
   journalctl -u betien-api -n 200 --no-pager
   ```
2. Revert the release commit or check out the last known-good SHA:
   ```bash
   git revert HEAD
   ```
3. If the release included a migration that must be reversed, run:
   ```bash
   alembic downgrade -1
   ```
4. Redeploy the known-good build:
   ```bash
   ./scripts/deploy_admin.sh
   ```
5. Reload Caddy and re-run smoke tests 1, 2, 5, and 10.

## Security verification

- `curl -I https://admin.betien.vn` includes HSTS, frame-deny, nosniff, and referrer-policy headers.
- Browser requests from origins other than `https://admin.betien.vn` do not receive permissive CORS headers.
- More than 100 `/api/admin/*` requests from one IP within one minute receive HTTP 429.
